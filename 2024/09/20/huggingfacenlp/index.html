Transformer models

transformer的功能及huggingface API

Transformer的功能有：

  1. 特征提取 feature-extraction：序列->特征向量
  2. 填空 fill-mask：有mask的序列->完整序列
  3. 实体识别 NER(named entity recognition)：序列->序列中实体的字典
  4. 回答问题 question-answering：问题&上下文->答案
  5. 情感分析 sentiment-analysis：序列->情感标签
  6. 概括 summarization：长文->短文
  7. 文本生成 text-generation：序列->扩展序列
  8. 翻译translation：序列->翻译序列
  9. 无微调分类 zero-shot-classification：序列->类别概率（这个类别在训练过程中没有给出，不需要微调就可以进行分类，所以叫zero-shot）

Transformer模型的历史和结构

Transformer模型的提出在2017年，原本是用于翻译任务的。
模型可以分成三类：

  1. GPT-like(auto-regressive)：自回归模型，仅decoder，仅从已有的信息推出后面的信息。
  2. BERT-like(auto-encoding)：自编码模型，仅encoder，从整个序列得到一个编码，优化编码的参数。
  3. BART/T5-like(sequence-to-sequence)：序列到序列模型，encoder-decoder。

“能不能看到整个序列”是encoder和decoder的主要差异。

区分几个术语：

  1. architecture 架构：指模型的骨架，里面有几层、每层是如何构建的。
  2. checkpoint 检查点：指模型的参数。
  3. model 模型：包含关于模型一切信息的概念。

Transformer模型是一个语言模型，而且是一个大模型。从头开始训练需要大量的数据、算力、时间，甚至还有巨大的碳排放。所以我们一般都是在预训练模型上进行微调。
但这也存在一些问题：不同的模型会因为投入的数据的语言、数据质量、数据分布等因素，存在一些bias（偏差，甚至是偏见）。比如在英语语料上训练的模型可能在其他语言上表现不好。
